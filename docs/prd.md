# 제품 요구사항 문서 (PRD): 위키 웹 애플리케이션

**문서 버전:** 1.0
**작성일:** 2025년 11월 3일

---

## 1. 개요

### 1.1. 프로젝트 목표
이 문서의 목표는 새로운 위키 웹 애플리케이션(이하 "gwiki") 개발을 위한 요구사항을 정의하는 것입니다. gwiki는 사용자가 개인적인 기록을 쉽고 빠르고 안전하게 기록하고 참조하도록 하기 위한 웹기반 도구입니다.

### 1.2. 배경
개인이 정보를 체계적으로 정리하고 필요할 때 빠르게 찾아볼 수 있는 중앙화된 공간이 필요합니다. 

### 1.3. 범위
이 문서는 gwiki의 초기 버전(1.0) 개발에 필요한 기능적, 비기능적 요구사항을 다룹니다.

---

## 2. 타겟 사용자

*   **개인 사용자:** 자신의 지식, 학습 노트, 개인 프로젝트 등을 체계적으로 관리하고 싶은 사용자.

## 3. 기능 요구사항

### 3.1. 핵심 기능: 페이지 관리
*   **페이지 생성 (Create):** 사용자는 새로운 위키 페이지를 생성할 수 있어야 합니다.
    *   페이지 제목과 본문으로 구성됩니다.
    *   본문은 마크다운(Markdown) 에디터를 사용하여 작성합니다.
*   **페이지 조회 (Read):** 사용자는 생성된 페이지를 조회할 수 있어야 합니다.
    *   URL은 `gwiki.com/pages/{페이지_제목}`과 같은 직관적인 형태여야 합니다.
    *   마크다운으로 작성된 본문은 HTML로 렌더링되어 보여야 합니다.
*   **페이지 수정 (Update):** 페이지를 생성한 사용자 또는 권한이 있는 사용자는 기존 페이지의 제목과 본문을 수정할 수 있어야 합니다.
*   **페이지 삭제 (Delete):** 페이지를 생성한 사용자 또는 권한이 있는 사용자는 페이지를 삭제할 수 있어야 합니다.

### 3.2. 사용자 인증
*   **회원가입:** 사용자는 이메일과 비밀번호를 사용하여 계정을 생성할 수 있어야 합니다.
*   **로그인/로그아웃:** 등록된 사용자는 시스템에 로그인 및 로그아웃할 수 있어야 합니다.
*   **사용자 프로필:** 사용자는 자신의 프로필 정보(예: 이름, email)를 관리할 수 있어야 합니다.

### 3.3. 검색
*   사용자는 키워드를 입력하여 위키 내의 모든 페이지 제목과 본문 내용에서 정보를 검색할 수 있어야 합니다.
*   검색 결과는 관련도 순으로 정렬되어야 합니다.

### 3.5. 네비게이션 및 구조
*   **사이드바:** 전체 페이지 목록 또는 사용자가 정의한 주요 페이지 링크가 사이드바에 표시되어 쉽게 탐색할 수 있어야 합니다.
*   **페이지 링크:** 사용자는 간단한 문법을 사용하여 다른 위키 페이지로 연결되는 링크를 쉽게 만들 수 있어야 합니다. 기본적으로 markdown 링크 형식으로 관리됩니다.

## 4. 비기능적 요구사항

### 4.1. 디자인 및 UX
*   **디자인:** 깨끗하고 미니멀한 디자인을 지향합니다. 사용자가 콘텐츠에 집중할 수 있도록 불필요한 시각적 요소를 최소화합니다.
*   **반응형 웹:** 데스크톱, 태블릿, 모바일 등 다양한 디바이스에서 최적화된 화면을 제공해야 합니다.
*   **사용성:** 모든 기능은 사용자가 별도의 학습 없이 직관적으로 사용할 수 있어야 합니다.

### 4.2. 성능
*   **페이지 로딩 속도:** 페이지 조회 시 2초 이내에 콘텐츠가 렌더링되어야 합니다.
*   **검색 속도:** 검색 결과는 1초 이내에 반환되어야 합니다.

### 4.3. 보안
*   사용자 비밀번호는 해시 처리하여 저장해야 합니다.
*   XSS(Cross-Site Scripting), CSRF(Cross-Site Request Forgery) 등 일반적인 웹 취약점으로부터 안전해야 합니다.

### 4.4. 기술 스택 (제안)
*   **프론트엔드:** React (반응형 UI 구현에 용이)
*   **백엔드:** Node.js (Express)
*   **Storage:** Filesystem
*   **버전관리:** Git

## 5. 가정 및 제약 조건

*   **가정:** 사용자들은 마크다운 문법에 대한 기본적인 지식이 있거나, 제공되는 에디터의 툴바를 통해 쉽게 배울 수 있습니다.
*   **제약 조건:** 초기 버전(1.0)은 파일 첨부 기능을 포함하지 않습니다. 텍스트 기반의 지식 관리에 집중합니다.
